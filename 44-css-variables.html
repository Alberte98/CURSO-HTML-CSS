1. Variables de CSS

Se definen con dos guines: --variable

1.1. Variables globales

- Pueden ser usadas en cualquier lugar de la página.

- Se definen dentro del selector :root


      --CSS
      --- Definicion Clase global

      :root {
        --margen: 100px
      }

      ---Llamada a clase global

      ul {
        margin-left: var(--margen); /*Variable global*/
      }

      --HTML
      Se aplica a todos los UL del HTML

1.2. Variables locales

Ejemplos de aplicación de variables:

Aplicado la clase tool-boxe lemento a elemento

  • Rojo y grande
  • Verde y mediano
  • Azul y pequeño
  • Chocolate y grande

Código:

      --HTML
      <ul>
        <li class="tool-box" id="rojo-grande">Rojo y grande</li>
        <li class="tool-box verde-mediano ">Verde y mediano</li>
        <li class="tool-box" att="azul-pequeño">Azul y pequeño</li>
        <li class="tool-box">Chocolate y grande</li>
      </ul>

Ejemplos de aplicación de variables:

Aplicado la clase toolbox a elemento padre

  • Rojo y grande
  • Verde y mediano
  • Azul y pequeño
  • Chocolate y grande

Código:

      --HTML
      <ul class="tool-box">
        <li id="rojo-grande">Rojo y grande</li>
        <li class="verde-mediano ">Verde y mediano</li>
        <li att="azul-pequeño">Azul y pequeño</li>
        <li>Chocolate y grande</li>
      </ul>

Contenido del CSS: 

        --CSS
  
        ---Definicion Clases contenedorea de variables locales
        .tool-box {
          --rojo: #ff0000;
          --verde: #00ff00;
          --azul: rgb(0, 0, 255);
          --chocolate: #955715;
  
          --pequeno: 20px;
          --mediano: 40px;
          --grande: 60px;
        }
  
  
        --Lamada a variables locales
        #rojo-grande {
          color: var(--rojo, violet); /*Variable local*/
          font-size: var(--grande);
        }
  
        .verde-mediano {
          color: var(--verde, violet); /*Variable local*/
          font-size: var(--mediano);
        }
  
        [att="azul-pequeño"] {
          color: var(--azul, violet); /*Variable local*/
          font-size: var(--pequeno);
        }
  
        li {
          color: var(--chocolate, violet);  /*Variable local*/
        }
  
        --HTML
        <ul>
          <li class="tool-box" id="rojo-grande">Rojo y grande</li>
          <li class="tool-box verde-mediano ">Verde y mediano</li>
          <li class="tool-box" att="azul-pequeño">Azul y pequeño</li>
          <li>Chocolate y grande</li>
        </ul>

1.3 Compatibilidad entre navegadores

Si un navegador como chrome es capaz de interpretar variables CSS, suponiendo que establezcamos una vairable para el color, podría ser utilizado así: 

        .paleta-color {
          --rojo: #ff0000;
        }

        .texto-rojo {
          color: var(--rojo, |color_respaldo|);
        }

En el ejemplo anterior si por algún motivo (escribir mal el nombre de la variable: --roja) se utilizará el color_respaldo

Cuando un navegador no interpreta variables css la solución será incluir la misma propiedad antes de la utilización de la variable

        .paleta-color {
          --rojo: #ff0000;
          --verde: #00ff00;
        }
        
        --Caso A: Sin color de respaldo.
        .texto-rojo {
          color: #f00;
          color: var(--roja);
        }
        
        --Caso B: Con color de respaldo
        .texto-verde {
          color: #0f0;
          color: var(--verde, #0f0);
        }

Es necesario seguir utilizando el color_respaldo ya que si se omite la variable o se ha escrito mal, no se mostrará ningún color (Caso A) ya que:.

  • El navegador leera la propiedad: 
    •           ...
          color: #f00;
          ...
  • Se interpretará la propiedad:
    •           ...
          color: var(--roja);
          ...

    - Pero ya que la función var() en la propiedad ha interpretada, pero el nombre de la variable ha sido mal escrita y no existe un color de respaldo, no se mostrará adecuadamente en el navegador.
